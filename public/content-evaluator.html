Bonario Content Evaluator — Technical Specification

Phân tích loại workflow
Pattern: Async Evaluation Pipeline
Đây là dạng Multi-step AI evaluation workflow — người dùng submit input, hệ thống chạy nhiều criteria song song (hoặc tuần tự), trả kết quả dần (progressive disclosure).
Submit → Queue → Process → Score Brand consistency engine Async + polling hoặc WebSocket Multi-criteria scoring

CRIT_001 · Cluster: All · Weight: 1.5x
Education Depth
"Sau khi đọc, người đọc rút ra được điều gì cụ thể về vật liệu hoặc cách ứng dụng trong không gian?"
FAIL nếu: chỉ mô tả cảm xúc, khen đẹp, hoặc quá ngắn để tạo giá trị
Bạn là chuyên gia đánh giá nội dung thương hiệu Bonario. Đọc caption sau và trả lời: 1. Người đọc học được điều gì cụ thể? (vật liệu, ứng dụng, kỹ thuật) 2. Nội dung có đi quá mức mô tả cảm xúc không? Caption: {caption_text} Cluster: {content_cluster} Trả về JSON: { "status": "PASS" | "FAIL", "evidence": "trích dẫn từ caption", "suggestion": "gợi ý sửa nếu FAIL" }
CRIT_002 · Cluster: Showcase, Education · Weight: 1.2x
Material Authority
"Caption có thể hiện kiến thức chuyên sâu về vật liệu — tên kỹ thuật, đặc tính, ứng dụng phù hợp?"
FAIL nếu: mô tả mơ hồ, không có thông tin kỹ thuật, chỉ dùng tính từ cảm tính
Đánh giá mức độ authority về vật liệu trong caption: - Có tên kỹ thuật không? (e.g. "họa tiết Jacquard", "bề mặt matte vinyl") - Có thông số sản phẩm không? (e.g. mã sản phẩm, bộ sưu tập) - Có giải thích ứng dụng không gian không? Caption: {caption_text} Trả về JSON: { "status", "evidence", "suggestion" }

Text–Visual Alignment
"Caption có mô tả đúng và bổ sung ý nghĩa cho hình ảnh layout đính kèm?"
FAIL nếu: caption mô tả thứ không có trong ảnh, hoặc ảnh và text mâu thuẫn nhau
Đây là caption: {caption_text} Đây là ảnh layout: [image_url_1], [image_url_2] Phân tích: 1. Caption có mô tả đúng nội dung trong ảnh không? 2. Có mâu thuẫn nào giữa text và visual không? 3. Frontend polling logic (pseudo-code)
// Frontend gọi sau khi submit async function pollEvaluation(evaluationId) { const INTERVAL_MS = 2500; const MAX_ATTEMPTS = 60; // timeout sau 2.5 phút let attempts = 0; return new Promise((resolve, reject) => { const interval = setInterval(async () => { attempts++; const res = await fetch(`/api/v1/evaluations/${evaluationId}/status`); const data = await res.json(); updateUI(data); // cập nhật progress, từng criteria card if (data.status === "COMPLETED" || data.status === "FAILED") { clearInterval(interval); resolve(data); } if (attempts >= MAX_ATTEMPTS) { clearInterval(interval); reject(new Error("Evaluation timeout")); } }, INTERVAL_MS); }); }
Lưu ý tích hợp cho CNTT
File upload
Upload ảnh lên S3/GCS trước, nhận presigned URL, gửi URL vào API — không gửi binary trực tiếp qua LLM API
LLM call strategy
Chạy các criteria song song (Promise.all) để giảm latency. Criteria cần ảnh chạy sau khi text-only xong
Rate limit
Giới hạn 10 evaluation/user/giờ. Queue với BullMQ hoặc tương đương
Brand config
Lưu brand_standard (voice, forbidden_phrases, keywords) trong DB, inject vào prompt khi gọi AI. Không hardcode
Audit trail
Lưu toàn bộ prompt + response AI vào evaluation_log cho mỗi criteria. Cần cho debug và tuning
Alternative: SSE
Thay polling bằng Server-Sent Events (EventSource API) để push update realtime — ít request hơn, UX tốt hơn
